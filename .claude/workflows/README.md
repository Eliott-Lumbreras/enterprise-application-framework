# Workflows (Fase 5)

Un workflow encadena varios skills/prompts en un orden fijo para una tarea completa, para que Claude no dependa de que el usuario recuerde cada paso. Los skills (Fase 2) definen el "como" del codigo, los prompts (Fase 4) definen "que" generar para una solicitud puntual, y los workflows definen "en que orden y cuando" se combinan para una tarea de mayor alcance.

| Workflow | Cuando se usa |
|---|---|
| new-project.md | Arrancar una aplicacion nueva desde cero dentro del framework |
| new-module.md | Agregar un modulo CRUD a un proyecto que ya existe |
| new-reporting-feature.md | Agregar un reporte + su dashboard de punto a punto |
| release.md | Pasar una version de dev/homologacion a produccion |
