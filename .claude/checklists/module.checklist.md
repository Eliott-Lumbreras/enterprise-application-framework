# Checklist maestro: antes de entregar un modulo

Este es el gate de SALIDA (post-codigo). El gate de ENTRADA (antes de generar codigo) es `design.checklist.md` (Fase 10) — si ese no paso, no debería existir codigo que revisar aqui todavia.

Repasar, en este orden, antes de marcar cualquier modulo como terminado:

- [ ] `security.checklist.md`
- [ ] `validation.checklist.md`
- [ ] `error-handling.checklist.md`
- [ ] `roles.checklist.md`
- [ ] `permissions.checklist.md`
- [ ] `audit.checklist.md`
- [ ] `logging.checklist.md`
- [ ] `performance.checklist.md`
- [ ] `testing.checklist.md`
- [ ] `swagger.checklist.md`

Si alguno falla, el modulo no esta listo, sin importar que el codigo "funcione" en una prueba manual rapida.
